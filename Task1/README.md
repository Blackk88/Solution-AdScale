## Целевые показатели

| Параметр | Сейчас | Через 3 мес | Через 12 мес |
|---|---|---|---|
| Throughput | 15 000 RPS | ≥ 18 000 RPS | ≥ 50 000 RPS |
| P95 bid response* | 110–120 ms | ≤ 80 ms | ≤ 80 ms |
| SLA доступности | — | без регресса от текущего | 99.9 % |
| Архитектура | монолит | монолит + bidding-сервис | микросервисы + event-driven |
| Развёртывание | ручное, 1 кластер | контейнеры, 1 регион | Kubernetes, multi-region |

\* Целевое значение определяется контрактным требованием DSP-партнёра — самым жёстким из применимых SLA.

## Ответы на ключевые вопросы

**Почему не оставить монолит?**
Под нагрузкой +20 % ожидаемый P95 превысит 150 ms (queuing theory: при ρ → 1 latency растёт нелинейно). SLA нового DSP-партнёра (≤ 80 ms) физически недостижим. Единственная PostgreSQL — SPOF и арена конкуренции OLTP/OLAP/write-stream. Подробности — [AS-IS.md](./AS-IS.md).

**Почему не big-bang переписывание на микросервисы?**
Нарушает срок 3 месяца, требует одномоментного переключения данных с остановкой системы (нарушает непрерывность бизнеса), штат 35 человек без SRE не способен одновременно эксплуатировать старую и новую систему 6+ месяцев. Подробности — [ADR-001](./adr/ADR-001-evolution-strategy.md).

**Почему Strangler Fig, а не модульный монолит как целевое состояние?**
Модульный монолит закрывает 3-мес фазу, но не масштабируется до 50 000 RPS на 12-мес горизонте — там нужно независимое масштабирование hot path и multi-region. Strangler Fig использует модульный монолит как **промежуточную форму** для ещё не выделенных доменов, эволюционируя к микросервисам по одному домену. Это позволяет каждой итерации приносить измеримое улучшение и не останавливать систему. Подробности и взвешенная матрица — [TO-BE.md §2](./TO-BE.md).

**Почему Bidding первым?**
Bidding — hot path с самым жёстким SLA (≤ 80 ms) и узкой предметкой. Каждая микрооптимизация (кэш, async stats, read-replica) даёт прямой эффект на SLA партнёра. Выделение Bidding также поднимает базовую платформу (API Gateway, Redis, Kafka, observability), которую переиспользуют остальные экстракции. Подробности — [ADR-002](./adr/ADR-002-bidding-service-first.md).
